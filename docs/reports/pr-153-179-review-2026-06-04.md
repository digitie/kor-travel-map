## 0. 총평

- 전체 품질은 높다. geocoding admin 라우터 제거(#153)는 docs·frontend·OpenAPI·
  test가 lockstep으로 정리된 **의도적·clean 제거**(dangling 없음). alembic
  0008~0011은 단일 head 연쇄·upgrade/downgrade 대칭·`models.py`/문서와 일치하며
  **ADR-012 위반(인덱스 술어 ST_Transform) 없음**. ADR-022 flat import 위반 없음.
- 그러나 **ADR-045 운영 형상(D-2 Dagster DB·daemon)**, **D-6 run-now 409 락**,
  **에러 envelope 계약**, **D-7 공개 응답 누출**, **offline-upload 업로드 크기
  상한** 등 운영·계약·보안에 직접 영향을 주는 HIGH 항목이 다수 존재한다.
- 교차 검출(중복 통합): D-2는 Dagster·Docker 리뷰 양쪽에서, D-6 락은 router·infra
  양쪽에서, resource engine leak은 Dagster 리뷰에서 검출됐다.

## 0.1 즉시 발견: main RED (merge skew 회귀)

리뷰 중 **main(`62e8a68`)이 RED** 임을 확인했다 — PR#178(`89e169b`)이
`settings.object_store_bucket` 기본값을 `krtour-map-files`→`krtour-map`으로
바꿨으나 `tests/unit/test_smoke_import.py::test_settings_default_values`의 기대값을
갱신하지 않아 3.11/3.12/3.13 unit이 모두 실패. PR별 CI는 green이었으나 자가 머지
배치에서 **merge skew**(개별 green, 결합 red)로 통과. 본 PR에서 stale 테스트를
실제 기본값(`krtour-map`, T-208b 오프라인 업로드 bucket)에 맞춰 정정했다.
→ 운영 시사점: **자가 머지 전 base 최신화 후 CI 재확인**(branch protection의
"require branches up to date") 또는 머지 큐 도입 검토(별도 운영 task).
