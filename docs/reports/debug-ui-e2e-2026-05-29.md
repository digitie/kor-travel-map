## 1. 환경 + provisioning

| 항목 | 값 |
|------|-----|
| Backend (WSL) | `./.venv/bin/python -m uvicorn krtour.map_debug_ui.app:app` :8087 |
| Frontend (WSL) | `next dev --port 8610 --hostname 0.0.0.0`, env `NEXT_PUBLIC_KRTOUR_MAP_DEBUG_UI_API=http://127.0.0.1:8087` |

| Windows Playwright | `@playwright/test` 1.60.0 + chromium (ms-playwright 캐시) |
| WSL2 forwarding | Windows `127.0.0.1:8610`/`:8087` → WSL 동일 포트 도달 확인 (양 서버 `--host 0.0.0.0`/`--hostname 0.0.0.0`) |

**provisioning (이번에 확립)**: 저장소에 npm workspace 루트가 없어 frontend가 한 번도 install된 적

신설하고, frontend의 `"@krtour/map-marker-react": "workspace:*"`(pnpm/yarn 문법, npm 미지원)를
npm 호환 `"*"`로 교체 → `npm install`(419 packages, github dep `maplibre-vworld#v0.1.0` 포함) 성공.
`@krtour/map-marker-react`는 런타임 미import(주석만)라 별도 빌드 불필요.

→ Windows 브라우저가 WSL frontend(:8610)를 렌더하고, 그 브라우저의 fetch가 WSL backend(:8087)의
`/debug/health`·`/debug/version`·`/debug/etl/*`에 도달(CORS 통과)해 정상 동작함을 **end-to-end로 검증**.

### 🐞 검출 + 수정한 실 버그
첫 e2e 실행에서 Next.js dev "Build Error" 오버레이가 떴다 (1 passed / 6 failed). 원인:

```
./src/app/etl/page.tsx — Syntax Error: Expected ';', '}' or <eof>
6 | * provider/dataset 선택 → "Preview" 버튼 → backend `/debug/etl/*/preview`
                                                              ^^ 이 `*/`가 블록 주석을 조기 종료
```

JSDoc 블록 주석(`/** ... */`) 안의 경로 `/debug/etl/*/preview`에 포함된 `*/`가 주석을 조기에 닫아
이후 텍스트가 코드로 파싱돼 빌드 실패. frontend가 한 번도 컴파일된 적 없어(node_modules 미설치)
PR#44부터 잠복했고, 본 e2e가 최초 컴파일로 검출. 주석을 `/debug/etl/{provider}/{dataset}/preview`로
수정 → `.next` 클린 후 재기동 → /etl 정상 컴파일 → **7/7 통과**.

(부수: WSL `/mnt/f`(NTFS)에서 next dev의 inotify hot-reload가 파일 수정을 놓쳐, 소스 수정 후엔
`.next` 삭제 + dev 서버 재시작이 필요했다.)

## 5. 재현 런북

### 5-1. (1회) Frontend provisioning — 이제 workspace 루트 commit됨
```bash
cd ~/dev/python-krtour-map && npm install   # 루트 workspace 전체 설치
```

### 5-2. (WSL) 두 서버 기동
```bash
# 터미널 A — backend
./.venv/bin/python -m uvicorn krtour.map_debug_ui.app:app --host 0.0.0.0 --port 8087
# 터미널 B — frontend
cd packages/krtour-map-debug-ui/frontend
NEXT_PUBLIC_KRTOUR_MAP_DEBUG_UI_API=http://127.0.0.1:8087 npx next dev --port 8610 --hostname 0.0.0.0
```

### 5-3. (Windows) Playwright 실행
```powershell
cd F:\dev\python-krtour-map\.e2e-win   # gitignored Windows 러너
npm install                            # @playwright/test (최초 1회)
Copy-Item ..\packages\krtour-map-debug-ui\frontend\e2e\*.spec.ts .\e2e\ -Force
$env:E2E_BASE_URL="http://127.0.0.1:8610"; npx playwright test
```
소스 수정 후엔 WSL에서 `rm -rf .next` + dev 서버 재시작(NTFS inotify 이슈).

## 6. 결론 + 후속

- **검증됨**: debug UI **backend + frontend 모두 WSL 기동**, **Windows Playwright e2e 7/7 통과** —
  frontend↔backend 실 연동(health/version/ETL providers/fixture preview)이 의도대로 동작.
- **수정됨**: frontend가 컴파일조차 안 되던 잠복 빌드 버그(`etl/page.tsx` 주석 내 `*/`) — e2e가 검출.
- **인프라**: npm workspace 루트 + Playwright 스위트 committed → 이후 `npm install` → 2서버 → e2e 재현 가능.
- **후속**: 지도 캔버스(maplibre-vworld) e2e는 `/features/*` 라우터 + 지도 컴포넌트 도입(후속 sprint) 후.
  frontend 컴파일을 CI/로컬 게이트에 추가하면 본류 버그를 조기 차단 가능(권장).
