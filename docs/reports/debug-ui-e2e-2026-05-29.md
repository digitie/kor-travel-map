# Debug UI 통합 검증 — WSL 기동 + Windows Playwright e2e (#117)

- **일자**: 2026-05-29
- **대상**: `packages/krtour-map-debug-ui` (FastAPI backend + Next.js frontend)
- **실행 모델 (사용자 지시)**: debug UI는 **WSL**에서 기동, Playwright는 **Windows**에서 실행.
- **결론 요약**:
  - ✅ **Backend (WSL :8087)** — 실 HTTP로 health/version/ETL providers/ETL fixture preview 전 경로 **검증 통과**.
  - ✅ **Playwright e2e 스위트** — `/`(홈)·`/etl` 페이지용 테스트 + config 작성 완료 (committed).
  - ✅ **Windows Playwright 런타임** — node v25 + Playwright 브라우저 캐시 설치 확인.
  - ⛔ **Frontend (WSL :8610) 라이브 e2e** — **provisioning 블로커**로 본 세션에서 미실행.
    원인: npm **workspace 루트 부재** → frontend의 `"@krtour/map-marker-react": "workspace:*"` 의존을
    `npm install`이 해석 불가 + `node_modules` 미설치. 사람 조치용 런북 §5 참조.

## 1. 환경

| 항목 | 값 |
|------|-----|
| Backend venv (WSL) | `./.venv/bin/python` (Linux), FastAPI/uvicorn import OK (11 routes) |
| Backend 포트 | `127.0.0.1:8087` (`uvicorn krtour.map_debug_ui.app:app`) |
| Frontend 포트 | `127.0.0.1:8610` (`next dev`, package.json `dev` script) |
| WSL node | v20.20.2 (nvm), npm 10.8.2 |
| Windows node | v25.9.0, npm 11.12.1 |
| Windows Playwright 브라우저 | `%USERPROFILE%\AppData\Local\ms-playwright` 존재 (설치됨) |
| API base URL (frontend→backend) | env `NEXT_PUBLIC_KRTOUR_MAP_DEBUG_UI_API`, 기본 `http://127.0.0.1:8087` |
| Backend CORS allow-origins | `http://localhost:8610`, `http://127.0.0.1:8610` (PR#68) |

WSL2 localhost-forwarding으로 Windows의 `127.0.0.1:8610`/`:8087`이 WSL의 동일 포트에 도달한다 →
Windows 브라우저가 WSL frontend(8610)를 열고, 그 브라우저의 fetch가 WSL backend(8087)에 닿는다.

## 2. Backend 라이브 검증 (WSL :8087) — ✅ 통과

`uvicorn krtour.map_debug_ui.app:app --host 127.0.0.1 --port 8087` 기동 후 실 HTTP 호출:

| 호출 | 결과 | HTTP |
|------|------|------|

| `POST /debug/etl/python-krex-api/krex_rest_areas/preview?source=fixture` | `count=2`, place FeatureBundle ("서산휴게소") | 200 |

`/debug/etl/providers` 카탈로그 (frontend `/etl` 드롭다운의 데이터 소스):

- **data.go.kr-standard**: `datagokr_cultural_festivals` (FeatureBundle)
- **python-kma-api**: `kma_short_forecast` / `kma_ultra_short_nowcast` / `kma_ultra_short_forecast` (WeatherValue), `kma_weather_alerts` (FeatureBundle)

정상적으로 FeatureBundle DTO(`feature_id`/`kind`/`coord`/`category`/`marker_*`)를 반환한다.

## 4. Frontend 라이브 e2e — ⛔ provisioning 블로커

본 세션에서 frontend(`next dev`)를 띄우지 못해 Windows Playwright **라이브 실행은 미수행**.

**근본 원인**: 모노레포에 **npm workspace 루트가 없다**.
- `packages/krtour-map-debug-ui/frontend/package.json`이 `"@krtour/map-marker-react": "workspace:*"`로
  `packages/map-marker-react`에 의존하나, 저장소 어디에도 `workspaces`를 선언한 루트 `package.json`이나
  `pnpm-workspace.yaml`이 없다 → `npm install`이 `workspace:` 프로토콜을 해석하지 못한다.
- frontend `node_modules` 미설치 (한 번도 install된 적 없음).
- `maplibre-vworld`는 github git 의존(`github:digitie/maplibre-vworld-js#v0.1.0`)이라 install 시 네트워크 필요.

이는 ADR-043("모노레포 내부 share")의 의도와 정합한 workspace 구성이 아직 코드에 없는 **구조적 갭**이며,
별도 PR에서 workspace 루트를 박는 것이 적절하다(구조 변경 + map-marker-react 빌드 검증 동반 → 본 e2e PR
범위와 분리).

## 5. 사람 조치용 런북 (라이브 frontend e2e 완료 절차)

### 5-1. (1회) Frontend provisioning — workspace 루트 확립
다음 중 하나로 `workspace:*`를 해석 가능하게 만든다(권장: 모노레포 루트 workspace).

```jsonc
// 예: 저장소 루트 또는 packages/ 에 package.json 신설
{
  "name": "krtour-map-monorepo",
  "private": true,
  "workspaces": ["packages/map-marker-react", "packages/krtour-map-debug-ui/frontend"]
}
```
그 후 루트에서:
```bash
npm install                       # workspace 전체 설치 (map-marker-react + frontend)
npm -w packages/map-marker-react run build   # (필요 시) 내부 패키지 빌드
```
대안: frontend `package.json`의 `workspace:*`를 `file:../../map-marker-react`로 임시 치환 후 install.

### 5-2. (WSL) 두 서버 기동
```bash
# 터미널 A — backend
cd ~/dev/python-krtour-map
./.venv/bin/python -m uvicorn krtour.map_debug_ui.app:app --host 127.0.0.1 --port 8087

# 터미널 B — frontend
cd ~/dev/python-krtour-map/packages/krtour-map-debug-ui/frontend
NEXT_PUBLIC_KRTOUR_MAP_DEBUG_UI_API=http://127.0.0.1:8087 npm run dev   # :8610
```

### 5-3. (Windows) Playwright 실행
```powershell
cd F:\dev\python-krtour-map\packages\krtour-map-debug-ui\frontend
npm run e2e:install   # 최초 1회 (chromium) — 브라우저 캐시 이미 있으면 빠름
npm run e2e           # http://127.0.0.1:8610 대상
npm run e2e:report    # HTML 리포트
```
다른 호스트면 `E2E_BASE_URL` env로 override.

## 6. 결론 + 후속

- **검증됨**: debug UI **backend** 전 경로(health/version/ETL providers/fixture preview)가 실 HTTP로
  의도대로 동작. ETL fixture preview는 DB 없이 provider 변환 함수를 실행해 정상 FeatureBundle 반환.
- **준비됨**: frontend **e2e 스위트 + config + scripts**가 committed — 위 런북으로 즉시 라이브 실행 가능.
- **블로커(사람)**: npm **workspace 루트 확립** (별도 PR 권장). 이후 §5-2/§5-3로 라이브 frontend e2e 완료.
- **참고**: 지도 캔버스(maplibre-vworld)는 아직 어떤 페이지에도 마운트되지 않음 → 지도 렌더 e2e는
  `/features/*` 라우터 + 지도 컴포넌트 도입(후속 sprint) 후 추가.
