## 0. 총평

- **전체 품질 높음.** 직전 리뷰의 T-RV-\* 하드닝이 운영/계약/보안 HIGH를 대부분
  닫았고, 이번 신규 feature 코드(정합성 F5~F8, T-200 batch gate)는 방어적으로
  잘 작성돼 있다. **이번 리뷰의 신규 지적은 전부 LOW**(관측 전용 WARN 케이스의
  count 의미/성능)다. HIGH/MED 신규 결함 없음.
- **검토 중 세운 2개 가설 risk는 ground truth로 모두 해소**(결함 아님):
  - F5 `provider_sync_state ⋈ provider_refresh_policies` LEFT JOIN의 행 fan-out
    우려 → `provider_refresh_policies` PK가 `(provider, dataset_key)` 복합 PK
    (`alembic 0009:305-306`)라 1:1 보장. **fan-out 불가**.
  - F7 score 스케일 불일치 우려(0~1 vs 0~100) → `dedup_review_queue.total_score`가
    `Numeric(5,2)` + CHECK `BETWEEN 0 AND 100`(`alembic 0005:65,88`)이고 F7은
    `score_pair()×100`(0~100)과 비교 → **스케일 일치**. baseline 회귀 비교 정상.
  - F6 시간 포맷 가정(`HHMM` regex) → DTO `opening_hours.py:5,34`가 `time`을
    `HHMM` pattern으로 강제 → **가정 일치**(거짓 음성 없음). `business_hours`/
    `opening_hours` 양쪽 + `special_days` 모두 커버.
- **batch gate(T-200) 안전성 양호**: `_quote_relation_name`이 MV 이름을
  `schema.view` SQL identifier로 검증(인젝션 차단), gate 분기(ERROR→mv_refresh
  차단+root failed / WARN·OK→refresh) 정확, `_normalize_uuid`로 child id 검증,
  consistency payload를 ERROR 차단 전에 기록.

## 2. 확인 후 이상 없음(혼동 방지용 기록)

- **F1/F2/F3**(정적 SQL): orphan source_record(LEFT JOIN ... IS NULL) / detail 누락
  (detail-bearing kind + 빈 JSONB, price·weather 제외) / CRS drift(ST_DWithin 0.01m
  tolerance) — 모두 ADR-018/012 정합, deleted_at 필터 정상.
- **F5**: PK 복합키로 join fan-out 불가(§0). `COALESCE(p.enabled, true)`로 정책 없는
  active cursor도 대상, `COALESCE(p.system_interval_seconds, :sla)`로 정책 우선. 정상.
- **F7**: 스케일 일치(§0). `primary_sources` CTE가 feature별 primary source 1건을
  `imported_at DESC` 결정적 선택, cross-provider(`psa.provider<>psb.provider`)만 검사.
  현재 feature 속성(name/category/coord) 재계산 vs 저장 baseline 비교 — 의도대로.
- **batch_dag**: 게이트/identifier 검증/UUID 검증/payload 기록 순서 정상(§0). 단일
  session·commit은 호출자 책임(consistency.py와 동일 계약).
- **T-208i offline 검증**: 입력이 byte 상한(T-RV-09) + `sample_size` row 상한으로
  bounded. `csv` 표준 모듈 사용.
